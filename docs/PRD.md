# Documento de Requisitos del Producto (PRD)

## Problema
Reconstrucción de un sistema de Historia Clínica desde Bubble a un stack tradicional.

## Objetivo (Done = ...)
Sistema operativo para médicos que permita gestionar pacientes, consultas y recetas bajo un entorno de despliegue controlado.

### Usuarios
* Método y Administrador.

### Módulos
* Autenticación
* Gestión de Pacientes (17 campos)
* Consultas
* Recetas
* Notificaciones

### Criterios de Aceptación
Cumplimiento de los 9 workflows críticos (Login, Registro Paciente, Receta, etc.).

## Configuración de Seguridad
Ver `config/feature-flags.json`.
